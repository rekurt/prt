## Что такое prt?

`prt` показывает, какие процессы занимают сетевые порты — в реальном времени, прямо в терминале. Интерактивный аналог `lsof -i` / `ss -tlnp` с подсветкой, фильтрацией и деревом процессов.

## Зачем prt?

Традиционные инструменты `lsof`, `ss` и `netstat` дают статический снимок, устаревающий в момент прочтения. `prt` предоставляет **живой, автообновляемый TUI** с отслеживанием изменений:

- **Видите соединения в реальном времени** — зелёный = новое, красный = закрывается
- **Мгновенно находите конфликты портов** — не нужно гадать с `lsof -i :8080`
- **Детектор подозрительных соединений** — аномалии автоматически помечаются `[!]`
- **Блокировка IP одной клавишей** — прямо из TUI через файрвол
- **Docker/Podman интеграция** — какой контейнер занимает какой порт
- **Трассировка syscall** на лету — strace/dtruss без выхода из TUI
- **Мониторинг пропускной способности** — RX/TX в заголовке
- **Настраиваемые алерты** — уведомления при открытии портов или превышении лимитов

## Возможности

### Живая таблица с отслеживанием изменений

Главное представление — сортируемая, фильтруемая таблица всех активных сетевых соединений. Колонки: порт, сервис, протокол, состояние, PID, процесс, пользователь. Новые соединения подсвечиваются **зелёным**, закрытые — **красным** на 5 секунд. Обновление каждые 2 секунды.

### База известных портов

Колонка `Service` сопоставляет порты с читаемыми именами — http (80), ssh (22), postgres (5432) и ~170 других. Можно дополнить или переопределить в `~/.config/prt/config.toml`:

```toml
[known_ports]
3000 = "my-app"
9090 = "prometheus"
```

### Возраст соединений

Каждое соединение отслеживает время создания (`first_seen`). ESTABLISHED старше 1 часа подсвечиваются жёлтым, старше 24 часов — красным. CLOSE_WAIT всегда красные, так как указывают на возможную утечку ресурсов.

### Детектор подозрительных соединений

Соединения проверяются на аномалии и помечаются `[!]`:

- **Non-root на привилегированном порту** — не-root процесс слушает порт < 1024
- **Скрипт на чувствительном порту** — Python, Perl, Ruby или Node.js слушают порт 22, 80 или 443
- **Root с исходящим на высокий порт** — root-процесс с established-соединением на удалённый порт > 1024

Фильтр: нажмите `/`, затем `!` — покажутся только подозрительные.

### Контейнеры Docker/Podman

Если запущен Docker или Podman, колонка `Container` показывает имя контейнера для каждого процесса. Колонка автоматически скрывается, если контейнеров нет. Разрешение выполняется пакетно через `docker ps` + `docker inspect` с таймаутом 2 секунды.

### Оценка пропускной способности

В заголовке отображается скорость сетевого трафика: `▼ 1.2 MB/s ▲ 340 KB/s`. На Linux читается `/proc/net/dev`, на macOS — `netstat -ib`. Скорости рассчитываются как дельты между циклами обновления.

### Дерево процессов

Нажмите `Enter` или `d` для открытия панели деталей, затем `1` — отобразится цепочка родительских процессов (например, `launchd → nginx → worker`).

### Разделы

`Tab` / `Shift+Tab` переключает между тремя разделами верхнего уровня.
Активный раздел подсвечен в шапке.

| Раздел | По умолчанию | Подвкладки (`[` / `]`) |
|--------|--------------|------------------------|
| **Соединения** | Таблица портов + нижняя панель Деталей (`Enter` / `d`) | — |
| **Процессы** | Детали выбранного процесса (CWD, CPU %, RSS, файлы, env, соединения, дерево процессов) | Детали ⇄ Топология |
| **SSH** | Сохранённые хосты и активные туннели в одном месте | Хосты ⇄ Туннели |

Панель **Деталей** под таблицей соединений — единый сводный вид: bind-тип,
интерфейс, удалённый адрес, состояние, cmdline, связанные порты процесса
и дерево родителей. Без вкладок.

Подвкладка **Топология** в Процессах рисует ASCII-дерево
`процесс → :локальный_порт → удалённый` для всего рабочего набора.

Везде, где есть скролл, работают `j`/`k` и `g`/`G`.

### Меню действий (`Space`)

Почти все операции над выбранной строкой собраны в один контекстный
popup, который открывается по `Space`:

- **Убить процесс** (есть и прямой шорткат `K`)
- **Копировать строку** (прямой шорткат `c`) / **Копировать PID**
- **Блокировать IP** — `iptables -A INPUT -s <IP> -j DROP` (Linux) /
  `pfctl -t prt_blocked -T add <IP>` (macOS). В статусной строке —
  команда отмены. Требует sudo.
- **Трассировать syscalls** — `strace -p <PID> -e trace=network -f`
  (Linux) или `dtruss -p <PID>` (macOS, нужен отключённый SIP или root).
  Повторный вызов отключает трассировку.
- **SSH-туннель** — открывает форму создания туннеля с пред-заполненными
  полями.

Меню показывает только применимые к строке действия — Блок и Туннель
скрыты, если у соединения нет удалённого адреса.

### Раздел SSH

`SSH` объединяет две подвкладки:

- **Хосты** — read-only список из `~/.ssh/config` плюс `[[ssh_hosts]]`
  из `~/.config/prt/config.toml`. `Enter` открывает форму туннеля с
  предзаполненным alias.
- **Туннели** — активные туннели с реальным статусом: 🟢 активен,
  🟡 запускается, 🔴 сбой (упавшие остаются в списке, пока вы их не
  обработаете).
  Клавиши: `n` новый · `e` правка · `K` убить · `r` рестарт ·
  `s` сохранить в конфиг.

Форма туннеля делает **inline-валидацию** (некорректные поля
подсвечиваются красным сразу при вводе), поддерживает **режим правки**
(Enter заменяет существующий туннель) и **защищает от случайного
закрытия** — `Esc` на непустой форме требует второго нажатия в течение
1.5 секунд.

### Правила алертов

Определите правила в `~/.config/prt/config.toml`:

```toml
[[alerts]]
port = 22
action = "bell"        # звуковой сигнал при новых SSH-соединениях

[[alerts]]
process = "python"
state = "LISTEN"
action = "highlight"   # подсветка строки жёлтым

[[alerts]]
connections_gt = 100
action = "bell"        # сигнал когда процесс превышает 100 соединений
```

Алерты срабатывают только на НОВЫХ записях (не каждый цикл обновления). Условия: `port`, `process`, `state`, `connections_gt`. Действия: `bell`, `highlight`.

### NDJSON-потоком

```bash
prt --json | jq '.process.name'
```

Выводит один JSON-объект на соединение за цикл обновления. Корректно обрабатывает SIGPIPE (без паник при пайпе в `head`). Без TUI — безопасен для скриптов.

### Watch-режим

```bash
prt watch 3000 8080 5432
```

Компактный не-TUI дисплей с UP/DOWN-статусом для конкретных портов. Звуковой сигнал (BEL) при смене состояния. ANSI-цвета при выводе в терминал, чистый текст при пайпе.

```
:3000 ● UP   nginx (1234)   since 14:32:05
:8080 ○ DOWN                 since 14:35:12
:5432 ● UP   postgres (567)  since 14:32:05
```

### Экспорт

```bash
prt --export json    # JSON-снимок всех соединений
prt --export csv     # CSV-снимок
```

### Мультиязычный интерфейс

Английский, русский, китайский. Приоритет определения:

1. Флаг `--lang en|ru|zh` (высший приоритет)
2. Переменная окружения `PRT_LANG`
3. Автоопределение по системной локали
4. Английский (по умолчанию)

Клавиша `L` переключает язык без перезапуска.

## Использование

```bash
prt                     # запустить TUI
prt --lang ru           # русский интерфейс
prt --export json       # экспорт в JSON
prt --export csv        # экспорт в CSV
prt --json              # NDJSON потоком в stdout
prt watch 80 443 5432   # компактный мониторинг портов
sudo prt                # запуск от root (все процессы)
```

## FAQ

### Как увидеть все процессы? Некоторые порты показывают "unknown".

Запустите `sudo prt` — без root ОС скрывает PID процессов других пользователей.

### Работает ли prt на Windows?

Пока нет. `prt` поддерживает **macOS** (10.15+) и **Linux** (с `/proc`). Поддержка Windows отслеживается в issue tracker.

### Чем prt отличается от `htop` или `btop`?

`htop`/`btop` — универсальные мониторы процессов. `prt` фокусируется на **сетевых соединениях и портах**: какой процесс на каком порту, жизненный цикл соединений, детекция аномалий и сетевые действия (файрвол, strace, SSH-туннели).

### Можно ли использовать prt в скриптах?

Да! `prt --json` для NDJSON-потока, `prt --export json|csv` для снимков, `prt watch <порты>` для простого мониторинга UP/DOWN.

### Безопасен ли prt для production?

`prt` — **инструмент только для чтения** по умолчанию. Деструктивные действия (kill, block IP, strace) всегда требуют подтверждения.
